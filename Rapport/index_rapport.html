01 · Présentation

À propos du projet

Le projet de cette année consiste à réaliser une application web qui met en relation les étudiants de l’IFRI souhaitant bénéficier ou offrir du mentorat académique et professionnel. Dénommé IFRI_MentorLink, chaque utilisateur crée un profil (compétences, disponibilités, filière, etc.) et peut publier ou rechercher des offres / demandes de mentorat. Un algorithme de matching basé sur la compatibilité des compétences, la proximité des filières et la compatibilité horaire propose automatiquement les combinaisons mentor-mentoré les plus pertinentes, tandis qu’une messagerie intégrée permet d’organiser et de suivre les sessions de mentorat.

07 · Conception

Architecture et Base de données

L'application a été conçue en respectant une architecture de type Client-Serveur et modulaire, conformément au cahier des charges. Nous avons utilisé le pattern MVT (Modèle-Vue-Template) offert par notre framework :

  • Frontend : HTML5, CSS3, JavaScript pour une interface dynamique et responsive.
  • Backend : Python propulsé par le framework Django.
  • Base de données : SGBD relationnel MySQL pour garantir l'intégrité des données.

Structure de la Base de Données :

  • User / CompteUtilisateur : Gestion de l'authentification sécurisée (mots de passe hashés) et du rôle (Mentor ou Mentoré).
  • Profil : Extension de l'utilisateur stockant la photo de profil, la filière, le niveau d'étude, les compétences et les lacunes.
  • Annonce : Centralise les offres et demandes avec la matière, le format (en ligne, présentiel) et les disponibilités horaires.
  • Matching : L'algorithme central qui gère les candidatures entre étudiants avec un système d'état (En attente, Accepté, Refusé).
  • Message : Gère l'historique des discussions pour la messagerie instantanée intégrée.
08 · Déploiement

Instructions de déploiement

Pour exécuter le projet IFRI MentorLink sur une machine locale, suivez ces étapes :

  • 1. Prérequis : Avoir Python 3.x, MySQL Server et Git installés sur la machine.
  • 2. Cloner le dépôt : Récupérer le code source depuis GitHub avec la commande git clone [URL_DU_DEPOT_GITHUB].
  • 3. Environnement virtuel : Créer un environnement isolé (python -m venv .venv) et l'activer.
  • 4. Dépendances : Installer les bibliothèques requises (Django, Pillow, mysqlclient) via la commande pip install -r requirements.txt.
  • 5. Base de données : Créer une base de données MySQL locale et configurer les identifiants de connexion dans le fichier mentorlink/settings.py.
  • 6. Migrations MySQL : Préparer et appliquer la base de données avec python manage.py makemigrations puis python manage.py migrate.
  • 7. Lancement du serveur : Démarrer le serveur de développement avec python manage.py runserver. L'application sera disponible sur http://127.0.0.1:8000/.
09 · Manuel d'utilisation

Comment utiliser MentorLink ?

L'expérience utilisateur a été pensée pour être intuitive :

  • Inscription : L'étudiant crée un compte en définissant son rôle principal. L'unicité de l'email est vérifiée.
  • Personnaliser son Profil : Rendez-vous dans l'onglet "Mon Profil". Cliquez sur l'icône du crayon (✏️) pour uploader une photo, et complétez votre filière et niveau d'études. Sauvegardez via le bouton "Enregistrer".
  • Définir ses Compétences : L'étudiant renseigne les matières qu'il maîtrise et ses lacunes dans l'onglet dédié.
  • Créer une annonce : Allez dans "Offres", cliquez sur le bouton "+ Nouvelle offre / demande", remplissez le formulaire dynamique et cliquez sur publier.
  • Trouver un match : Depuis l'onglet "Offres", parcourez les annonces de la communauté et cliquez sur le bouton "Répondre" pour postuler.
  • Accepter un mentorat : Vérifiez vos "Notifications". Vous y verrez les demandes reçues. Cliquez sur "✅ Accepter" pour valider la connexion avec l'étudiant.
  • Messagerie : Une fois le match accepté, allez dans l'onglet "Messages". Le contact apparaîtra dans la liste, débloquant ainsi le chat pour organiser vos sessions d'études.
